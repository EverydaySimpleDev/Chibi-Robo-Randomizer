================================================================================
 CHIBI-ROBO (.us) PER-LEVEL SCRIPT GUIDE
================================================================================
 What the labels in each stage script are, and what each level's script contains.


--------------------------------------------------------------------------------
 LABEL TYPES (first - so the per-level lists make sense)
--------------------------------------------------------------------------------
   evt_<name>_<id>   NAMED event handlers - the meaningful "functions" of a
                     level. The name tells you what it is (an NPC, an item, a
                     loading zone, a stain...). These are what this guide lists.
   loc_<N>           AUTO-GENERATED jump labels (branch targets inside a routine).
                     There are thousands; individually they're just "go here"
                     points with no standalone meaning, so they are NOT listed
                     per-level. Read the code around a loc_ to see its role.
   sub_<N>           Internal subroutines (called via `run *sub_N`). Mostly
                     generic helpers specific to one routine.
   lib_<N>           Library functions - these exist ONLY in globals.us and are
                     shared by every stage (called via `lib N.w`). They are
                     documented separately in lib_functions.txt, NOT per-level.

 So "per level" the real content is: the ENTRY POINTS + the evt_* handlers.


--------------------------------------------------------------------------------
 ENTRY POINTS (every house stage declares the same set)
--------------------------------------------------------------------------------
   .prologue  *evt_prologue     one-time intro for the room
   .startup   *evt_startup      runs EVERY time you enter the room (door setup,
                                NPC placement, spawns - this is where the door
                                flags are read)
   .dead      *evt_dead         player-death handler
   .pose      *evt_pose         pose-action handler
   .time_cycle *evt_time_cycle  runs on a DAY/NIGHT change (calls lib 73); NOT
                                on every room load
   .interact <objId>, *evt_...  one per interactive object in the room


--------------------------------------------------------------------------------
 COMMON evt_ NAME PREFIXES (decoder)
--------------------------------------------------------------------------------
   map_jump_box       a room loading zone (a warp trigger). Its handler does the
                      transition and `warp <room>`.
   plug_start         a power outlet (plug in to recharge).
   plug_dokodakadoor  a Chibi-Door (the warp-network doors / "dokodaka door").
   house_r_chibi_h_*  Utilibot / Chibi-House warp objects (main/warp/door/dai).
   npc_<who>          a character interaction. Names:
                        tonpy=Telly  mama=Mom  papa=Dad  jenny=Jenny  tao=Tao
                        bb=Sophie  et_1=Aliens  army_1=Sarge  army_2=Free Rangers
                        army_3/4/5=ranger variants  hock=Captain Plankbeard
                        fafa=Sunshine  sun_mida=Peekoe  keath=Drake Redcrest
                        zobin=Mort  peets=Princess Pitts  t_rex/t_track=Dinah
                        frog/dry_frog=Frog  bird=Bluebird  deca_robo=Giga-Robo
                        consentman=Mr. Prongs  spider_1/2/3=Spydorz
                        funky_f/funky_f_3=Funky Phil/Freaky Phil  primo=Primopuel
   item_<name>        a pick-up (the <name> is the item, e.g. frog_ring, cracker,
                      deka_denchi=Giga-Battery, papa_yubiwa=Dad's ring).
   yogore_<kind>      a STAIN/mess to clean:
                        water=water  shimi=dirt  hito=footprints  hun=bird poop
                        oil=oil  kureyon=crayon (brush) / kureyon_k=crayon
                        (squirter)  toy=Dinah prints  ff=Funky sweat  cook_o=honey
                        h_s/h_k=Drain stains
   living_happy_box   a breakable Happy-Point box.   sa_seal = Sunshine sticker.
   living_asiato      Tao pawprint.   takarabako = treasure chest.
   test_box_/test_dust_  debris / boxes.

 Room IDs used by `warp`:
   0 Debug  1 Kitchen  2 Foyer  3 Basement  4 Jenny's Room  5 Chibi-House
   6 Bedroom  7 Living Room  9 Backyard  11 Drain  16/17 Chibi-House  (14 intro)

--------------------------------------------------------------------------------
 HOW TO INSPECT ANY SPECIFIC HANDLER
--------------------------------------------------------------------------------
   1. In the stage's .us, find the declaration:  .interact <id>, *evt_<name>
   2. Jump to the label `evt_<name>:` and read down to its `return`.
   3. loc_<N>: labels inside it are just internal branches; follow them as
      needed. sub_<N> are `run`-called helpers; lib N.w are global (see
      lib_functions.txt).
   For the door-specific routines and flags, see door_flags.txt.
