HOW TO READ AN ENTRY
====================
  Each flag takes two lines:

      19  CONFIRMED  Giga Robo sticker / Giga Robo state (in All-Stickers block)
                      set 1 / read 104  -  stage01, globals, stage02, ...
      |   |          |                     |          |
      |   |          |                     |          +- files that touch it
      |   |          |                     +------------ how many times set vs read
      |   |          +---------------------------------- what the flag means
      |   +--------------------------------------------- confidence (see below)
      +------------------------------------------------- the flag number

  The list is in ascending flag-number order, broken into labelled BANDS so you
  can scan by topic. To find a specific flag, just search for it (e.g. "  718 ").


CONFIDENCE TAGS
===============
  CONFIRMED   From an explicit in-script comment, the sticker list, or a known
              convention. Trust these.
  INFERRED    Read off a named event label or surrounding code. Usually right,
              occasionally imprecise.
  GUESS       Weak / ambiguous - only generic labels or no context. A hint, not
              a fact (often just part of a bulk reset block).

  Totals:  52 CONFIRMED   .   558 INFERRED   .   368 GUESS


WHAT A FLAG IS
==============
  Flags are global on/off game-state bits.  `set flag(N.d), 1.w` turns flag N
  on;  `if eq(flag(N.d), 1.d) ...` reads it.  The .b / .w / .d suffixes are just
  integer sizes (byte / word / dword) and carry no meaning.

  Two related, NON-flag states show up in some descriptions for context:
    atc(N)    unlocked attachment:  1 = copter, 2 = blaster, 3 = radar
    cur_suit  the costume / suit Chibi-Robo is currently wearing


EVENT-LABEL DECODER  (the main signal behind INFERRED meanings)
===============================================================
  evt_item_<name>        a collectible / story item  (the <name> IS the item)
  evt_map_jump_box_<n>   a room / scene transition  (loading zone / door warp)
  evt_npc_<name>_<n>     an NPC conversation / event  (dialogue progress)
  evt_living_* / evt_backyard_* / evt_chibi_house_*    location-specific events
  evt_prologue / startup / dead / pose    intro / boot / death / pose cutscenes
  lib_<n>                shared library subroutine  (weak signal)
  loc_<n> / sub_<n>      auto-generated labels  (no semantic meaning)


NOTABLE FLAG BLOCKS  (these are the bands in the list below)
============================================================
  add(1700, itemId)   "item itemId has been collected/received" - the
                      item-received markers (~1700-1834). They use a
                      computed index, so they do NOT appear as numbers below.
  100 - 156    Battery system (100 = enabled, 101-137 level-up bits,
               119 = LV2 milestone, 138-156 level-up notifications).
  160 - 208    lib_74 NPC / event-progress dispatch table (per-NPC bits).
  209 - 299    "shown current suit to this NPC" reaction bits (suit x NPC).
  604 - 607    Four collectible panels / parts, set together.
  701 - 799    lib_275 day/time reset block (zeroed each day/time reset).
               Mostly unknown; 701 = ladder-bot / saving-family,
               716 / 717 / 718 / 1013 = Chibi-Door panels.
  908 - 928    Toy soldiers rescued / captured (win-condition group).
  1030 - 1048  Chibi-House door used, per stage.
  1100 - 1119  Stage 11 (drain) scripted-sequence step markers.
  1200 - 1224  Player position / footstep trigger markers.
  1501 - 1520  First-time "wearing suit X" effects + suit-shown reactions.
  1606 - 1611  Tracks the current suit / costume worn (lib_264).
